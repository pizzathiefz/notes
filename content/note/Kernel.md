## 커널 밀도 추정

- Kernel Density Estimation(줄여서 KDE)
	- 알려지지 않은 확률 분포를 추정하기 위함

![[assets/kernel/pdf.png|400]]

- 확률 밀도 함수(probability density function; PDF)
	- 연속형 변수인 $X$를 받아서 특정 값($x=a$)일 확률 밀도(density)값을 돌려주는 함수
	- 주의할 점은 여기서 밀도인 $\text{PDF}(x)$는 **확률 값이 아님**
		- 내가 맨날 만드는 피자 크기가 정확히 15인치일 확률은 어떻게 될까? 어떤 기계가 와도 정확히 ‘15인치’를 만들 수 없음 (도우의 지름이란 변수의 정의 자체가 연속적인 실수값이기 때문), 따라서 $x=15$일 확률은 0
		- 하지만 대충 ‘15인치 언저리’일 가능성을 상대적으로 나타낼 수는 있고 밀도 함수로부터 ‘15인치에서 16인치 사이’일 확률도 계산할 수 있음 -> 그림에서 a와 b 사이의 면적, 즉 PDF의 적분값을 계산하면 변수가 특정 구간의 값을 가질 확률이 됨

가지고 있는 데이터를 가지고 확률 밀도 함수를 추정할 때 가능한 방법은

1. 특정 분포를 따른다고 가정하고, 주어진 데이터에 맞게 파라미터만 추정하기 = parametric (e.g. 정규분포라고 가정하고 그 파라미터인 평균, 분산만 찾기)
2. 사전 가정 없이 주어진 데이터만 가지고 추정하기 = non-parametric

![[assets/kernel/histogram.png|375]]
- 히스토그램 그리기도 non-parametric하게 데이터만 가지고 분포를 볼 수 있는 방법 중 하나
	- 단 불연속적이고, bin(막대/데이터를 카운트하는 기준 구간 길이)의 크기를 어떻게 설정하느냐에 따라 모양도 달라진다는 문제 
- 이 히스토그램을 저 검은색 선처럼 연속적인 선으로 바꾸려면? 이때 커널 함수를 사용
- 커널 함수 $K$는 다음 2가지 특징을 만족하는 non-negative 하고 적분 가능한 함수여야 함
	- 적분 값이 1이다. ( $\int_{-\infty}^{+\infty} K(u) du = 1$ )
	- 원점을 기준으로 대칭이다. ($K(-u) = K(u)$   $\forall u$)
- 해당 조건을 만족하는 커널 함수에는 다음과 같이 많은 종류가 있음

![[assets/kernel/kernel function.png|425]]

이미지 [출처](https://en.wikipedia.org/wiki/Kernel_(statistics))

- $K$를 다음과 같이 사용하여 히스토그램 대신 smooth한 밀도함수 $f(x)$ 를 얻는 것이 바로 KDE 기법

$$
\hat{f_h} (x) = \frac{1}{n} \sum ^n _ {i=1} K_h (x-x_i) = \frac{1}{nh} \sum^n _{i=1} K (\frac{x-x_i}{h})
$$

- 수식을 살펴보면, 각 $x$값에 대해 다른 모든 $x$값과의 거리를 $h$라는 bandwith 파라미터로 조절된 커널 함수 $K$에 넣고, 이것을 모든 $x$값에 더한 다음 데이터의 개수인 $n$만큼 나눠줌 
	- bandwith $h$는 커널함수가 얼마나 뾰족해야 할지 조절하는 값


![[assets/kernel/histogram to kde.png|425]]

이미지 [출처](https://en.wikipedia.org/wiki/Kernel_density_estimation)

- 오른쪽에서 커널 함수들(빨간색)이 많이 겹치는 구간일수록 데이터가 몰려 있다고 볼 수 있으므로 최종적으로 추정된 밀도 함수(파란색)에서는 확률 밀도 값이 높게 표현되는 걸 볼 수 있음

- 거리 값을 넣는 이유: 각 $x$와 멀리 떨어진 다른 $x_i$ 에 대해서는 적은 값을 돌려주게 되도록 (위의 다양한 커널 함수 생김새를 보면 원점에 가까울수록 큰 값, 멀어질수록 작은 값)
    - 즉 5 근처의 확률 밀도 값에 5와 멀리 떨어진 데이터들은 적게 기여하고 가까운 데이터들은 높은 가중치로 기여하도록 → 즉 어떻게 보면 $x$와 $x_i$의 가까운 정도(유사도)를 돌려주는 함수
- 데이터 개수로 나눠주는 이유:  확률 밀도 함수는 적분값이 1(모든 확률의 합은 1)이어야 하는데 커널 함수를 데이터 개수만큼 $n$번 더했으므로 마지막에 $n$으로 나눠줌


> **정리 : KDE에서 커널이란 비모수적 추정을 할 때 불연속적 분포를 smooth하게 표현하기 위해 사용되는 함수**

## 커널 회귀

- 보다친숙한 선형 회귀(linear regression)의 경우, X와 Y의 관계가 선형일 것이라는 가정 하에 그 선형 관계의 양상을 나타내주는 파라미터를 데이터를 기반으로 추정 = parametric
- 반면 커널 회귀와 같은 비모수적인 방법은 별도의 가정 없이 주어진 데이터만 가지고 Y값을 예측하는 함수를 만들고자 함 
	- 잘 된다면, 다음과 같은 비선형적인 관계도 잘 표현할 수 있다는 장점이 있음

![[assets/kernel/non-linear.png|300]]

- 컨셉은 KDE와 비슷함 = KDE에서 각 데이터포인트별로 나와 가까이 있는 다른 데이터들이 높은 값으로 반영되도록 커널 함수를 사용한 것처럼, 이번에도 Y를 예측할 때 **가까운 값들에 높은 가중치를 둔 가중평균을 통해** 예측

$$
y_t = \frac{1}{n} \sum^n_{i=1} w_iy_i
$$

- 이때 가중치는 다음과 같이 커널 함수 $K$를 이용해서 계산

$$
w_i = \frac{ K(x_t-x_i)}{  \sum ^n _ {i=1} K( x_t-x_i)}
$$

- 앞서 언급한 대로 $x_t$와 거리가 멀어질수록 높은 값을 돌려주는 커널 함수를 사용하게 되므로, $x_t$와 가까운 데이터포인트일 수록 $y_t$에 높은 가중치를 두어 합하게 됨
	- 이와 같은 방식을 통해 KDE에서처럼 저런 smooth한 예측 함수 선을 만들어낼 수 있음


> **정리: 커널 회귀에서 커널이란 가까운 데이터들의 가중평균을 통해 예측 변수를 예측할 때 가중치 값으로 사용되는 함수**

## 커널 SVM

![[assets/kernel/svm.png|250]]

- Support Vector Machine(SVM)
	- 분류 과제에서 쓰일 수 있는 지도학습 모델 중 하나로, 분류를 위한 최적의 결정 경계선을 찾고자 함 
	- SVM이 ‘최적’이라고 생각하는 경계선은 경계선과 가장 가까이 있는 데이터들로 이루어진 서포트 벡터와 경계선 사이의 거리(마진)가 최대화되는 경계선
	- 이 경계선을 찾는 문제는 라그랑지안 승수 $\alpha$에 대해 다음과 같은 목적 함수로 정의

$$
\max L_D(\alpha_i) = \sum^n_{i=1} \alpha_i - \frac{1}{2} \sum^n _{i=1} \sum^n_{j=1}\alpha_i\alpha_j y_iy_jx_i^Tx_j
$$

- 이때 SVM의 한계 중 하나는 결국 경계’선’을 그어야 한다는 것인데, 선을 어떻게 그어도 완벽한 분류가 불가능한 데이터를 만난다면?


![[assets/kernel/kernel-svm.png|550]]

이미지 [출처](https://towardsdatascience.com/the-kernel-trick-c98cdbcaeb3f)

- 좌측처럼 1차원인 상태에서는 하나의 선으로 두 개의 그룹을 분류할 수 없었지만, 이를 2차원으로 변환하자 최적의 선을 찾을 수 있는 게 보
- 이처럼 어떤 맵핑 함수 $\Phi$ 가 있어서, 우리가 가진 원래 데이터를 선을 그어서 분류가 가능한 새로운 데이터로 바꿔줄 수 있다면 어떨까

$$
\max L_D(\alpha_i) = \sum^n_{i=1} \alpha_i - \frac{1}{2} \sum^n _{i=1} \sum^n_{j=1}\alpha_i\alpha_j y_iy_j \Phi(x_i)^T\Phi(x_j)
$$

- 문제는 이러한 맵핑으로 원 데이터를 고차원으로 변환하는 것, 그리고 그것을 내적하는 것 ($\Phi(x_i)^T\Phi(x_j)$ )에 상당한 연산 비용이 든다는 것
- 이럴 때 커널 함수를 사용


$$
K(x_i, x_j) = \Phi(x_i)^T\Phi(x_j)
$$

- [머서의 정리](https://en.wikipedia.org/wiki/Mercer's_theorem)에 따르면, 커널 함수 $K$가 특정 조건을 만족하기만 하면  $K(x_i, x_j) = \Phi(x_i)^T\Phi(x_j)$ 인 맵핑 함수 $\Phi$ 가 존재한다고 함 
- 따라서 우리가 $K$를 이용한다면 $\Phi$를 실제로 찾을 필요도 없고 무엇보다 매우 비용이 큰 내적 계산을 건너뛸 수 있다!  = **커널 트릭**
- 또 재미있는 점은, $K$는 두 개의 서로 다른 벡터의 내적이므로 수식상으로 코사인 유사도를 계산하는 것과 아주 비슷하게 생겼음
	- 즉 KDE에서와 마찬가지로 이 함수는 일종의 유사도 함수로 볼 수 있는 것
	- 특히 커널이 될 수 있는 조건 중 하나가 대칭성, 그리고 positive-definite한 성질이 있는 점도 이런 해석과 어긋나지 않음


>정리:  **커널 SVM에서 커널이란 선형 분류가 가능하도록 데이터를 변환할 때 고차원 맵핑 및 내적에 필요한 연산량을 감소시키기 위한 기교로서 사용되는 함수**




## 마무리

- 커널이란 그냥 **특정 조건을 만족하는 변환용 함수**이고, 어떤 목적으로 사용하느냐에 따라서 커널이 뭐냐는 질문에 따라 다른 답을 할 수 있겠지만, 일반적으로 만족하는 특성 상 어느 정도는 그 사용 방식에서 지속되는 공통점이 있음(smoothing 목적으로 쓰인다거나, 가까움의 표현 유사도로서 해석될 수 있다거나)
